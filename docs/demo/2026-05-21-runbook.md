## Step 1 — Slack incoming webhook (1회)

1. https://api.slack.com/messaging/webhooks → "Create a Slack App from scratch" → 워크스페이스 선택.
2. "Incoming Webhooks" 활성화 → "Add New Webhook to Workspace" → 채널 `#dsapm-demo` 선택.
3. URL 복사:
   ```bash
   export SLACK_WEBHOOK_URL='https://hooks.slack.com/services/XXX/YYY/ZZZ'
   ```
4. (선택) 즉시 검증:
   ```bash
   curl -sS -X POST "$SLACK_WEBHOOK_URL" \
     -H 'Content-Type: application/json' \
     -d '{"text":"DS-APM 데모 채널 검증 OK"}'
   ```
   기대 응답: `ok`. 채널에 "DS-APM 데모 채널 검증 OK" 메시지가 보이면 성공.

## Step 2 — SigNoz + OTel 데모 스택 기동

`DS_APM_AI_GENERATOR=mock` 와 `DS_APM_AI_MOCK_FIXTURE_DIR=/var/ds-apm/mock-fixtures` 는 이미 compose override에 박아 두었다. mock fixture 디렉터리는 호스트의 `pkg/ruler/aigenerator/mockaigenerator/testdata` 를 read-only 바인드 마운트한다 (`.omc/demo/docker-compose.dsapm-override.yaml`). **별도 shell export 불필요.**

cd <HOME>/workspace/signoz-demo

  -f sources/opentelemetry-demo/docker-compose.yml \
  -f sources/opentelemetry-demo/docker-compose.signoz.override.yml \

## Step 3 — SOP 3종 시드 (`org-1`)

```bash
cd <REPO_ROOT>
for f in docs/demo/sop_pay.json docs/demo/sop_cart.json docs/demo/sop_ad.json; do
  echo "POST $f"
  curl -sS -X POST http://localhost:8080/api/v2/ds/sop/documents \
    -H 'Content-Type: application/json' \
    -H 'X-SigNoz-Org-Id: org-1' \
    -d @"$f"

```

```bash
curl -sS http://localhost:8080/api/v2/ds/sop/documents \

"SOP-AD-001"
```

## Step 6 — 데모 진행 (LIVE)

> **v0.1 한계:** alertmanager dispatcher가 아직 AI generator를 자동 호출하지 않는다 (`pkg/ruler/aigenerator/dispatchhook/`은 standalone 컴포넌트로만 빌드되어 있고 dispatcher wire-in은 v0.2 작업). 그래서 시연 흐름은 **(a) Preview 엔드포인트로 AI 응답을 미리 받고 → (b) flagd flip 으로 알람 발화 → (c) 시연자가 AI 응답 본문을 Slack thread 에 붙여넣음** 의 3박자다.
>
> 청중에게는: "이번 v0.1에서는 운영자가 한 번 더 Preview를 호출해 paste 합니다. v0.2에서는 같은 응답이 자동으로 Slack 본문에 박혀 나옵니다" 로 설명.

각 시나리오 진행 시 항상:

```bash
# (a) AI 응답 미리 받기
curl -sS -X POST http://localhost:8080/api/v2/ds/ai/strategy/preview \
  -H 'Content-Type: application/json' \
  -H 'X-SigNoz-Org-Id: org-1' \
  -d '<JSON>' | jq '.headline, .firstActions, .hypotheses'
```

`<JSON>` 은 시나리오별로 아래 6a/6b/6c 본문 참조.

## Troubleshooting

| 증상 | 원인 후보 | 빠른 점검 |
|---|---|---|
| AI Preview 응답이 `mock-*-001` 이 아님 (e.g. `language=ko-KR` 만 있고 headline 없음) | `DS_APM_AI_GENERATOR` 가 mock 으로 안 들어감 | `docker compose exec signoz env | grep DS_APM_AI` |
| AI Preview 가 5xx | mock fixture dir 가 컨테이너 안에서 안 보임 | `docker compose exec signoz ls /var/ds-apm/mock-fixtures` |
| Slack 알람 안 옴 | webhook URL 오타 / 채널 권한 / 룰 not firing | SigNoz UI → Alerts → 해당 룰 → "Recent Firings" 확인. Step 1 검증 curl 재실행 |
| flagd 명령이 `exec: flagd not found` | flagd 컨테이너 entrypoint 다름 (버전 차이) | `docker compose ... exec flagd /flagd --help` 로 실제 경로 확인 후 명령어 보정 |
| SOP 시드 POST 가 404 | API path 가 `/api/v2/ds/sop/documents` 가 아닐 수 있음 | `pkg/apiserver/.../ds_router.go` 또는 SigNoz UI Network 탭에서 실제 path 재확인 |
