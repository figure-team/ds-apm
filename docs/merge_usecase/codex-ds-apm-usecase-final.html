병합 기준

유스케이스는 “사용자가 시스템으로 달성하려는 목표”만 다룬다. 장애별 절차, curl smoke test, fixture 설명은 docs/sop/로 분리했다.

등록되지 않은 에러 패턴은 UC3의 하위 흐름으로 다룬다. 관련 SOP를 찾고, runbook이 없으면 AI draft를 만들되 자동 저장/실행하지 않는다.

액터

운영자 / SRESOP, Runbook, AI 설정, 알림 대응 자산을 만들고 incident 중 대응 절차를 확보한다.
뷰어등록된 알림, SOP, Runbook, AI 전략 이력을 조회한다.
알림 시스템alert rule 발화와 dispatch 흐름을 통해 incident context를 만든다.
LLM ProviderAI 전략과 Runbook draft를 생성한다.
SigNoz 관측 기능trace, log, metric, dashboard, service map으로 진단 근거를 제공한다.
Pilot SOP 소스managed markdown 같은 외부 SOP 원천을 제공한다.

안전 원칙

  • AI draft는 자동 저장되지 않는다.
  • Draft는 자동 실행되지 않는다.
  • Approved runbook도 시스템이 직접 실행하지 않는다.
  • Deprecated runbook은 기본 목록에 노출하지 않는다.
  • 잘못된 입력은 저장 전에 거절한다.
