# Story 7.6: 반복 알림 억제·무시 처리

Status: planned

## Story
**운영자**는 동일 예외의 반복 알림에 시달리지 않으며, 무시 처리한 예외는 다시 알림받지 않는다. *(목적: 알림 피로 없이 신규 신호에만 집중하기 위해)*

## Acceptance Criteria
1. 동일 fingerprint 재발은 억제 윈도우(기본 24h) 내 재알림 없이 count·last_seen만 갱신된다 (FR-CF7.6)
2. dismiss 처리된 시그니처는 알림·분석 모두 영구 수행되지 않는다 (count만 갱신)
3. dismissed→new 직접 전이는 금지된다
4. 주기당 신규 시그니처가 임계(N건, 설정 가능) 초과 시 묶음 알림 1건으로 축약된다
5. 운영자는 UI에서 시그니처 목록(상태·발생수·서비스·최근 발생)을 조회하고 dismiss할 수 있다

## Tasks / Subtasks
- [ ] 억제 윈도우 판정 + count·last_seen 갱신 (Story 7.2 제외 로직 연동) (AC: 1)
- [ ] REST `POST /api/v2/ds/exceptions/{fingerprint}/dismiss` + 상태 전이 검증 (AC: 2,3)
- [ ] 상태머신 금지 전이 테스트 (AC: 3)
- [ ] 폭주 축약 (묶음 알림) (AC: 4)
- [ ] REST `GET /api/v2/ds/exceptions` (필터·페이징) (AC: 5)
- [ ] frontend 미등록 예외 목록 페이지 + dismiss 액션 (i18n en/ko) (AC: 5)
