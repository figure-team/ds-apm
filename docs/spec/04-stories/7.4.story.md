# Story 7.4: 저장소 연결·코드 전송 통제

Status: planned

## Story
**관리자**는 분석에 사용할 git/svn 저장소 연결과 소스코드 외부 전송 허용 여부를 설정으로 통제한다. *(목적: 코드 자산의 외부 노출을 조직 정책으로 통제하기 위해)*

## Acceptance Criteria
1. 관리자는 org 단위로 저장소 목록(name/type(git|svn)/url/서비스명 매핑/브랜치)을 등록·수정·삭제할 수 있다 (FR-CF7.4)
2. 자격증명은 secretbox 암호화 저장되고 조회 응답에 비노출된다(write-only)
3. 로컬 미러가 주기(기본 10분) 동기화된다 — git `clone --mirror`+fetch, svn checkout+update, 위치 `var/vcs/{org}/{repo}`
4. `include_code_context=false`면 LLM 요청에 소스코드가 포함되지 않고 보고서에 "코드 컨텍스트 제외"가 명시된다
5. 미러 실패는 감사 기록되고 분석은 "코드 컨텍스트 불가"로 degrade된다 (분석 자체는 진행)
6. 미러 디렉터리는 org별 격리된다

## Tasks / Subtasks
- [ ] `ds_vcs_config` 모델·store (org당 1행, 마이그레이션 081 일부) (AC: 1)
- [ ] 자격증명 secretbox 암호화 (ds_ai_config 패턴 재사용) + 응답 마스킹 (AC: 2)
- [ ] 미러 매니저 (`pkg/vcs/`) — git/svn 어댑터, 주기 fetch, org별 디렉터리 (AC: 3,6)
- [ ] REST `GET/PUT /api/v2/ds/vcs/config` (기존 handler 컨벤션, authz·감사) (AC: 1,2)
- [ ] include_code_context 스위치를 분석 워커에 전달 (Story 7.3 연동) (AC: 4)
- [ ] 미러 실패 degrade 경로 + 감사 (AC: 5)
- [ ] frontend 설정 화면 (기존 AI 모듈 설정 페이지 패턴, i18n en/ko)
