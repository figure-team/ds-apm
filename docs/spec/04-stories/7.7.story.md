# Story 7.7: 분석 실패에도 발생 알림 보장 (fail-open)

Status: planned

## Story
**운영자**는 코드 분석·AI가 실패해도 신규 예외 발생 알림 자체는 누락 없이 받는다. *(목적: AI 부정확·장애가 silent drop보다 나쁘지 않도록 — 불변 원칙)*

## Acceptance Criteria
1. 1차 알림은 분석 파이프라인과 독립적으로(동기·즉시) 전송된다 — 분석 단계의 어떤 실패도 1차 알림을 지연·차단하지 않는다 (FR-CF7.7)
2. LLM 장애/quota 초과 시 2차 알림은 생략되되, 실패가 감사 기록되고 시그니처 상태에 표시된다
3. 스택트레이스 파싱 실패 시 로그 기반 분석으로 degrade된다 (보고서에 명시)
4. VCS 미러 미설정/실패 시 "코드 컨텍스트 불가"로 degrade된다 (분석 진행)
5. 1차 알림 채널 실패는 기존 DLQ(CF-5)로 보존·재발송된다

## Tasks / Subtasks
- [ ] 스캐너의 1차 알림과 분석 큐 적재의 독립성 보장 (알림 실패가 큐 적재를 막지 않음, 역도 동일) (AC: 1)
- [ ] LLM 실패 분기 — 감사 + 시그니처 상태 표시 (AC: 2)
- [ ] degrade 분기 2종 (파싱 실패→로그 기반, 미러 실패→컨텍스트 불가) + 보고서 명시 (AC: 3,4)
- [ ] DLQ 경유 확인 (synthetic alert도 기존 terminal failure 처리 적용) (AC: 5)
- [ ] 인수 테스트 (godog, 한글 스텝): UJ-5 골든패스 + 분석 실패 분기 + dismiss + 등록 후 재발 흡수
