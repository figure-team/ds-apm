# Story 1.2: 등록한 SOP의 보관·매칭

Status: done (as-built)

## Story
**관리자**는 등록한 SOP가 무결하게 보관·매칭됨을 신뢰한다. *(목적: 한 번 등록하면 신뢰할 수 있도록)*

## Acceptance Criteria
1. 등록한 SOP 본문이 영속되고 재조회 시 무결성 검증을 통과해 그대로 반환된다 (FR-CF1.2)
2. 최신 버전 조회 시 가장 최근 승인 버전이 반환된다
3. 본문 무결성: `sha256:` checksum · 256 KiB 상한 · markdown 형식 · `safeDisplayURL`

## Tasks / Subtasks
- [x] `SOPStore` 인터페이스 정의 (Upsert/Get/GetLatest/List/Delete) (AC: 1,2)
- [x] SQL 구현체 `sqlsopstore` (bun ORM, `(org_id,sop_id,version)` 키) (AC: 1)
  - [x] 마이그레이션 078 `ds_sop_documents` 테이블 (AC: 1)
- [x] 무결성 검증 구현: checksum·256 KiB·markdown magic·`safeDisplayURL` (AC: 3)
- [x] `GetLatest` version DESC 정렬 (zero-pad 버전 가정) (AC: 2)
