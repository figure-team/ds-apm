# Story 11.6: 분석 실패에도 알람 무영향 (fail-open)

Status: done

## Story
**운영자**는 코드 분석·CLI가 실패·타임아웃·비용 초과해도 알람 전달이 영향받지 않으며, 실패가 감사·상태로 기록됨을 보장받는다. *(목적: AI 부정확·장애가 silent drop보다 나쁘지 않도록 — 불변 원칙)*

## Acceptance Criteria
1. 디스패치 훅은 additive·non-blocking이며 분석은 비동기다 — 훅은 unbound 분기 입력을 변경 없이 즉시 반환한다 (FR-CF11.6)
2. run이 `run_timeout`(기본 5분)을 초과하면 CLI는 종료되고 status=timeout으로 기록되며 lease·reaper가 슬롯을 회수한다
3. CLI/파싱 실패 시 status=failed|unparseable로 기록되고 raw 출력이 보존된다
4. 모든 실패는 감사 이벤트로 기록된다(best-effort, `auditor.Audit`)
5. 어떤 실패 분기에서도 알람 전달 경로는 영향받지 않는다 (silent drop 0)

## Tasks / Subtasks
- [x] 엔진 종료 상태 전이(`runOne` → done/failed/timeout/unparseable) (AC: 2,3)
- [x] per-run 하드 ceiling — `run_timeout` ctx-cancel + `WaitDelay`, claude `--max-budget-usd` (AC: 2)
- [x] 파서 tolerant 처리(malformed → unparseable, raw 보존) (AC: 3)
- [x] 감사 이벤트(coderca audit) (AC: 4)
- [x] 디스패치 훅 additive 보장 — unbound 분기 무변경 반환(설계 §5.1) + e2e fail-open 인수(`integration_test.go` `TestUJ5FailOpen`) (AC: 1,5)
