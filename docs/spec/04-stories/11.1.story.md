# Story 11.1: 트리거 게이트 + AI 코드 RCA 보고서

Status: done

## Story
**운영자**는 매칭 SOP가 없고(unbound) 이상(anomaly)이 감지된 알람에 대해, AI 코딩 에이전트가 코드베이스를 탐색해 만든 근본원인+수정 제안 보고서를 받는다. *(목적: 따라야 할 Runbook이 없는 신종 장애에서 코드 근거 RCA를 자동으로 받기 위해)*

## Acceptance Criteria
1. 게이트는 *기능 ON ∧ unbound ∧ anomaly ∧ 심각도 임계 이상 ∧ 서비스→저장소 매핑* 일 때만 후보를 통과시킨다 (FR-CF11.1)
2. anomaly 신호가 없으면 발화하지 않는다 (skip=no_anomaly, fail-closed)
3. 게이트 통과 시 영속 `coderca_run`(status=queued)이 적재되고 훅은 즉시 반환한다
4. 워커가 CLI 에이전트를 checkout 안에서 read-only로 구동해 근본원인·수정 제안·신뢰도·기준 커밋이 담긴 RCAResult를 생성한다 (status=done)
5. 프롬프트는 에이전트에게 (a)에러 시그니처에 맞는 코드 경로 탐색, (b)근본원인 가설, (c)수정 제안(diff/단계), (d)신뢰도·한계, (e)분석한 기준 커밋 에코를 지시한다

## Tasks / Subtasks
- [x] `SkipReason` 열거 + 게이트 판정(feature_off/no_anomaly/below_severity/no_repo_mapping 등) (AC: 1,2)

- [x] RCA 프롬프트 빌더 `BuildPrompt` + `RCAContext`/`EvidenceCollector`(Noop v1) (AC: 5)
- [x] CLI 구동 어댑터 연동(clirunner) + 결과 파서(rcaresult) (AC: 4)
- [x] 디스패치 훅 unbound(=Missing) 분기에서 `trigger.Maybe(ctx, orgID, labels, annotations)` 호출 + `CodeRCATrigger` 주입(setter) (설계 §10·§11)
