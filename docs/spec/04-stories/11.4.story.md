# Story 11.4: HITL 수정 제안·read-only 샌드박스

Status: done

## Story
**운영자**는 AI가 제안한 수정이 자동 적용되지 않고 검토용 제안으로만 전달되며, 에이전트가 코드에 **읽기 전용**으로만 접근함을 보장받는다. *(목적: AI 오탐이 코드 변경·장애 확대로 이어지지 않도록 — HITL 불변)*

## Acceptance Criteria
1. CLI 호출은 read-only 플래그가 정확히 포함되어 조립된다 (FR-CF11.4)
   - codex: `codex exec -s read-only -C <checkout> -m <model> --json`
   - claude: `--allowed-tools "Read,Grep,Glob" --disallowed-tools "Bash,Write,Edit,WebFetch,WebSearch" --max-budget-usd <cap>` (`--dangerously-skip-permissions` 미사용)
2. fake-binary 하니스에서 쓰기 시도는 거부된다
3. 에이전트 자격증명 준비(codex `CODEX_HOME`/`auth.json`, claude `CLAUDE_CODE_OAUTH_TOKEN`)는 기존 어댑터 방식을 재사용하되 `cmd.Dir=<checkout>`·minutes-scale 타임아웃을 추가한다
4. per-run checkout이 에이전트의 유일한 쓰기 표면이며 실행 종료(실패·타임아웃·kill 포함) 시 제거된다
5. 산출물의 수정 제안(ProposedFix)은 검토용으로만 전달되고 어떤 저장소에도 자동 적용되지 않는다

## Tasks / Subtasks
- [x] read-only 호출 조립 `BuildArgs` (codex/claude 각각 정확한 플래그) (AC: 1)
- [x] 에이전트 인증 env 준비(codex/claude) (AC: 3)
- [x] Runner 실행 + codex/claude 출력 캡처 (AC: 1,3)
- [x] fake-binary 통합 테스트(읽기전용 플래그 단언 + write-rejection + timeout) (AC: 2)
- [x] 프로세스 그룹 실행(`Setpgid`)·`Pdeathsig`·`WaitDelay` (고아 방지) (AC: 4)
- [x] `RCAResult.ProposedFix` = suggestion only (자동 적용 경로 부재) (AC: 5)
