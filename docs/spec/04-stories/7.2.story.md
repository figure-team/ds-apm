# Story 7.2: 미등록 신규 예외 자동 인지·1차 알림

Status: planned

## Story
**운영자**는 Alert Rule에 등록되지 않은 신규 예외가 발생하면 시스템으로부터 자동 알림을 받는다. *(목적: 등록된 현상만 기다리지 않고 모르는 장애를 놓치지 않기 위해)*

## Acceptance Criteria
1. Alert Rule이 커버하지 않는 신규 예외 시그니처가 텔레메트리에 출현하면 스캐너 주기 내 인지된다 (FR-CF7.2)
2. 시그니처는 정규화(예외 타입+최상위 앱 프레임+서비스, 메시지 가변부 제거)되어 동일 원인 재발이 한 시그니처로 묶인다
3. 신규 시그니처는 `ds_exception_signatures`에 등록되고(status=new), 1차 알림 성공 시 notified로 전이된다
4. 1차 알림은 synthetic alert(라벨 `ds_unregistered_exception=true`)로 기존 dispatcher를 경유한다 — PII 마스킹·DLQ·감사 동일 적용
5. 기존 rule 커버·linked SOP·dismissed·억제 윈도우 내 시그니처는 알림되지 않는다

- [ ] known 제외 판정 (rule 커버 / linked_sop_id / dismissed / 억제 윈도우) (AC: 5)
