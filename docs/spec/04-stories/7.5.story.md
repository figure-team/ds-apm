# Story 7.5: 보고서의 SOP/Runbook 자산화 (HITL)

Status: planned

## Story
**운영자**는 분석 보고서를 검토한 뒤 신규 SOP/Runbook으로 등록하며, 등록물은 승인 전 효력이 없다. *(목적: 미등록 예외의 대응 지식을 개인 경험이 아닌 플랫폼에 자산화하기 위해)*

## Acceptance Criteria
1. 운영자는 보고서에서 "SOP/Runbook으로 등록"을 실행할 수 있다 — 신규 SOP 생성 또는 기존 SOP 선택 (FR-CF7.5)
2. Runbook 초안은 **draft 상태로만** 생성된다 — 자동 승인 없음, 승인은 기존 상태머신(draft→approved)
3. 등록 시 시그니처에 linked_sop_id가 기록되고 status=registered로 전이된다
4. registered 시그니처의 재발은 신규 예외로 재알림되지 않는다 (CF-1 경로로 흡수 전제)
5. 등록 행위는 감사 기록된다
6. Alert Rule 자동 생성은 수행하지 않는다 (open item — 안내 문구만)

## Tasks / Subtasks
- [ ] REST `POST /api/v2/ds/exceptions/{fingerprint}/register` (신규/기존 SOP 분기) (AC: 1)
- [ ] 보고서 Runbook 초안 → 기존 Runbook draft 변환·임베드 (SOP store 재사용) (AC: 2)
- [ ] 시그니처 linked_sop_id·status 전이 (AC: 3)
- [ ] 스캐너 제외 판정에 registered 반영 (Story 7.2 연동) (AC: 4)
- [ ] 감사 기록 (AC: 5)
- [ ] frontend 등록 모달 — SOP 선택/생성 + Runbook 초안 미리보기·수정 (`RunbookForm`·`RunbookDraftFromError` 패턴 재사용, i18n en/ko) (AC: 1,2,6)
