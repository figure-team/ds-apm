# Story 11.2: 저장소·서비스 매핑·기능 토글 설정

Status: done (store·secretbox · HTTP/FE는 통합 seam)

## Story
**관리자**는 분석 대상 저장소와 `(org, 서비스명)→저장소[+subpath]` 매핑, org/서비스 단위 기능 on/off와 비용 임계값을 설정으로 통제한다. *(목적: 코드 자산 접근과 분석 비용을 조직 정책으로 통제하기 위해)*

## Acceptance Criteria
1. 관리자는 저장소(git_url/default_branch/enabled/자격증명)와 서비스→저장소 매핑을 등록·수정·삭제할 수 있다 (FR-CF11.2)
2. 기능은 org/서비스 단위 on/off이며 기본 **OFF**(opt-in), 비용 임계값(일일 한도·동시성·쿨다운 등)은 per-org 오버라이드 가능하다
3. 자격증명은 secretbox(AES-256-GCM)로 ciphertext 저장되고 조회 응답에 평문이 반환되지 않는다(write-only)
4. 암호화 키(`DS_APM_AI_CONFIG_ENCRYPTION_KEY`)가 없으면 비공개 저장소 자격증명 저장·사용을 거부한다 — 평문 저장 금지(fail-closed). 공개·무자격 저장소만 허용
5. 전 테이블은 org 단위 격리된다
