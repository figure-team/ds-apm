### 주요 의사결정 (확정)

| 항목 | 결정 |
|---|---|
| WBS Lv2 골격 | Component-oriented (6 컴포넌트, Phase는 부록) |
| 상세 에러 케이스 2건 | UC-002 (Channel → DLQ → Replay) + UC-003 (LLM auth fail-open) |
| 언어 톤 | 한국어 본문 + 영문 ID/스키마 (`shall` → "~해야 한다") |
| Gherkin 키워드 | 영문 (`Given/When/Then`) — godog 호환 |
| arc42 버전 | v9.0 (2025-07) |
| 변경 용이 구조 | atomic 1파일=1개념, stable ID frontmatter, single source of truth (`.md` → `.html` 빌드) |
| 팀장용 5종 톤 | 한국 SI 표준 (외래어 한국어 풀이 병기, 격식체 ~합니다) |

### DS-APM 정체 (오해 정정 적용 완료)

- DS-APM은 **SigNoz의 fork도 별도 시스템도 아닙니다**.
- 같은 단일 Go 바이너리(`cmd/community/`) 안에 박힌 확장 모듈.
- SigNoz Alertmanager의 dispatcher hot path를 wrapping해서 SOP/AI/DLQ 단계를 끼워넣은 구조.
- **외부 HTTP 호출은 OTLP 인풋 / LLM API / 5채널 출구 3 지점뿐**. 그 외 모든 흐름은 동일 프로세스 내 함수 호출.
- 이 정체는 모든 산출물의 다이어그램·본문에 일관 적용됨 (SigNoz outer subgraph + DS-APM inner subgraph).
