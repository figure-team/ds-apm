공통 기반 모듈 착수 예정
공통 규약·테넌트 격리 정책·감사 기록을 다섯 컴포넌트에 제공.
보장감사 누락률 0% (모든 SOP조회·AI요청·발송 영속 기록)
보장감사 파일 50 MiB 자동 회전
보장타 조직 SOP는 '존재 없음'으로 응답(누설 금지)
보장자격증명 외부 노출 0(자동 차단)
한계테넌트 격리가 라벨 필터 수준 — DB 행단위 보안 아님. 외부 다수 고객사엔 강화 필요(내부 단일 무방)
확인SOP 1건 조회 시 감사 파일에 '누가·언제·무엇·결과' 1줄 추가. sink 초기화 실패해도 부팅은 진행.

상세: 기능명세서 · 일감 WBS-1.0

SOP 그라운딩 서비스 착수 예정
알람 라벨(예: signoz_pilot_sop_id)로 등록된 SOP를 자동 매칭. markdown·버전·승인(draft→approved).
보장명시적 라벨 매칭 — 결과 결정적(추론 오류 없음)
보장SOP 본문 256 KiB 상한
보장disabled SOP는 '사용 불가' 명시
한계명시적 라벨만 — 의미기반 벡터 검색 v0.1 미포함
한계90일 경과 SOP 자동 보류(staleness)
확인라벨 매칭되는 approved SOP가 있으면 5초 내 찾아 AI 단계로 전달. 타 조직 조회는 '찾을 수 없음'.

상세: 기능명세서 · 일감 WBS-1.1

AI 초안 매니저 착수 예정
매칭 SOP+라벨을 외부 LLM에 보내 대응 초안(가설·첫 조치·안내문)을 받음. 호출 한도·시간·라이선스 검사, 실패해도 발송 안 멈춤(fail-open).
보장발송 경로에서 1초 내 결판 — 초과 시 SOP 원문만으로 진행
보장모든 첫 조치에 '운영자 승인 필요' 강제(자동 실행 주장 거부)
보장한도 소진·라이선스 사유 감사 기록
한계AI 초안은 승인 전 자동 실행 안 됨(판단은 사람)
한계외부 LLM 의존(단 fail-open)
한계fail-closed 옵션은 후속 결정
확인LLM이 401/429면 1초 내 SOP 원문으로 흘려보내고 '초안 없음' 표시 + SRE 메타 알람 1건.

상세: 기능명세서 · 일감 WBS-1.2

알림 디스패처 착수 예정
Alertmanager 발송 경로에 SOP·AI·DLQ 단계 삽입, 5채널 동시 발송. 채널별 포맷 자동 변환(Block Kit/Adaptive Card), 22 표준 필드.
보장5채널 동시 — 수동 누락률 ~5%→0%
보장발송 p95 30초 이내(승인 시간 제외)
보장한 채널 실패해도 나머지 계속, 실패는 DLQ 흡수
보장잘못된 템플릿 변수 등록 단계 차단
한계MS Teams 승인 버튼(Action.Submit) 미작동 — OpenUrl만
한계Slack 대화형 승인은 후속
확인결제 5xx → SOP 매칭 → 3채널 30초 내 도달. Slack 실패 시 로그·DLQ 기록하고 나머지 계속.

상세: 기능명세서 · 일감 WBS-1.3

PII 마스킹 필터 착수 예정
외부로 나가기 전 이메일·KR 전화·32자+ 시크릿·Bearer/JWT/API키·URL 민감 파라미터를 22 필드 일괄 가림.
보장4종 패턴 누출률 0%
보장URL user:password@host 항상 제거
보장채널 어댑터 호출 이전 완료 — 미정제 값 외부 못 나감
한계입구 단일 지점만 — OTel Collector 단 이동 권장(README 경고)
한계가림율 메트릭·카테고리 확장 후속
확인'ops@example.com'·'010-1234-5678' 포함 시 발송 메시지는 [redacted-email]·[redacted-phone]로 치환.

상세: 기능명세서 · 일감 WBS-1.4

DLQ 재처리 서비스 착수 예정
발송 실패분을 원본·사유와 함께 DLQ(영속 파일) 보관. 재전송 시 replay ledger로 중복 차단. 재기동 후에도 중복 0.
보장재시도 실패 후에도 원본 손실 0건
보장DLQ 50 MiB 자동 회전
보장DLQ 쓰기 실패해도 발송 안 멈춤(best-effort)
보장같은 이벤트ID 재전송 2번째 자동 skip
한계HMAC 서명 정책 미정 — 재전송 위변조 검증(추후 D-1)
한계중복키 fingerprint만 — 채널별 독립 ledger 후속
한계재전송 UI/CLI는 범위 밖(후속)
확인Slack 5xx 시 DLQ에 채널·이벤트ID·사유 1줄. 같은 ID 재전송 2번째는 skip 기록.

상세: 기능명세서 · 일감 WBS-1.5
